📌 คู่มือการสแกนสินค้า

รูปแบบที่รองรับ:

barcode8852673000311
barcode,qty8852673000311,50
location,barcode,qtyH11,8852673000311,50
SKU10012345 (ระบบหา Barcode หน่วยเล็กสุดให้)
SKU,qty10012345,10
location,SKU,qtyA01,10012345,10

ขั้นตอนการนับสต็อก:

1 เลือกสาขา (SRC / KKL / SSS)
2 อัปโหลด R01.102 และ R05.106 (ครั้งแรกเท่านั้น ข้อมูล sync ขึ้น Cloud อัตโนมัติ)
3 สแกน Barcode หรือ SKU → กด Enter ทุกครั้ง
4 อัปโหลด R16.104 (ยอดขายระหว่างนับ) ก่อนกด Confirm
5 กด ✔ Confirm → ระบบเปรียบเทียบกับ System Qty อัตโนมัติ
6 รายการที่ขึ้น Audit → กด ยืนยันจำนวน ในแถวนั้นเพื่อเปลี่ยนเป็น Audit Check

Cloud Sync & Backup:

☁️ ดึงข้อมูลดึงข้อมูลล่าสุดจาก Cloud (กรณีเปิดจากเครื่องอื่น)
💾 Backupสำรองข้อมูลเป็นไฟล์ JSON ลงเครื่อง
📂 Restoreกู้คืนข้อมูลจากไฟล์ JSON ที่ Backup ไว้

สถานะผลลัพธ์:

✅ Passจำนวนตรงกับระบบ
⚠️ Auditจำนวนไม่ตรง รอ Auditor ยืนยัน
✅ Audit CheckAuditor ยืนยันจำนวนแล้ว
❓ Unknownไม่พบใน R01/R05 — นับแยกใน Not in System
❓ สาเหตุที่ขึ้น Unknown
(1)SKU ถูกลบออกจาก R01.102 แต่ Barcode ยังอยู่ใน R05.106
(2)SKU เปลี่ยนรหัสใน R01 แต่ R05 ยังใช้รหัสเก่า
(3)Barcode ไม่มีในระบบเลย (สินค้าใหม่/ยังไม่ลงทะเบียน)
